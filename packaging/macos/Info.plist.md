# macOS `Info.plist` notes (VoiceFlow)

The `.app`'s `Info.plist` is **generated by PyInstaller's `BUNDLE` step**, not a
committed file. The keys are passed via `info_plist={...}` in
`packaging/voiceflow.spec` (the `if IS_MAC:` block). This document records *what*
is set and *why*, so the source of truth stays the spec.

## Keys set by the BUNDLE step

| Key | Value | Why |
|---|---|---|
| `CFBundleIdentifier` | `com.voiceflow.app` | Stable bundle id; TCC permission grants and the app's Keychain/defaults bind to this + the code signature. Changing it resets all granted permissions. |
| `CFBundleName` / `CFBundleDisplayName` | `VoiceFlow` | App name in Finder / menu bar / permission prompts. |
| `CFBundleShortVersionString` / `CFBundleVersion` | `1.0.0` | Marketing + build version (keep in sync with `src/voiceflow/__init__.py`). |
| `NSMicrophoneUsageDescription` | "VoiceFlow records your microphone to transcribe speech locally on your device." | **Mandatory** — without this string the OS kills the app the instant it touches the mic. This is the text shown in the mic permission prompt. |
| `LSUIElement` | `true` | Runs as a **menu-bar agent** (no Dock icon, no app menu). VoiceFlow lives in the tray/menu bar, matching the Windows tray behavior. |
| `LSMinimumSystemVersion` | `11.0` | Big Sur+ (matches the pyobjc / arm64 wheel floor). |
| `NSHighResolutionCapable` | `true` | Retina-crisp UI. |
| `NSHumanReadableCopyright` | MIT note | About box / Finder Get-Info. |

## Permissions: plist vs. runtime TCC (important)

macOS needs **three** distinct things for VoiceFlow to work; only the first is a
plist key:

1. **Microphone** — `NSMicrophoneUsageDescription` (plist, above). Prompted
   automatically on first mic access.
2. **Input Monitoring** (`kTCCServiceListenEvent`) — to *listen* for the global
   hotkey. **Runtime TCC grant**, no plist key. The user enables it in
   System Settings → Privacy & Security → Input Monitoring.
3. **Accessibility** (`AXIsProcessTrusted` / `kTCCServicePostEvent`) — to *post*
   the synthesized **Cmd+V** paste. **Runtime TCC grant**, no plist key. Enabled
   in System Settings → Privacy & Security → Accessibility.

`voiceflow.platform.macos` detects (2) and (3) via `AXIsProcessTrusted` and a
listen-event probe, and `voiceflow.ui.first_run` opens the exact Settings pane:

```
open "x-apple.systempreferences:com.apple.preference.security?Privacy_Accessibility"
open "x-apple.systempreferences:com.apple.preference.security?Privacy_ListenEvent"
```

## Why signing + notarization is non-negotiable here

TCC permission grants are keyed to the **code signature / identity**. An
unsigned or ad-hoc-rebuilt `.app` does **not** reliably trigger the prompts and
loses any previously-granted permissions on every rebuild. Therefore the release
pipeline (PRODUCTION_PLAN.md §5.2) **must**:

1. Build the `.app` via `BUNDLE` (every inner `.dylib`/`.so` signable).
2. `codesign --force --deep --options runtime --timestamp --entitlements packaging/macos/entitlements.plist -s "Developer ID Application: … (TEAMID)" dist/VoiceFlow.app`
3. Package into a `.dmg` (`hdiutil … -format UDZO`).
4. `xcrun notarytool submit … --wait` (notarytool, **not** the retired altool).
5. `xcrun stapler staple dist/VoiceFlow.dmg` (you cannot staple a `.zip`).
6. Verify: `spctl --assess --type open --context context:primary-signature -v`.

## Editing the plist

To change a plist value, edit the `info_plist={...}` dict in
`packaging/voiceflow.spec`, **not** this file. If a key must be patched *after*
PyInstaller writes it (rare), use `/usr/libexec/PlistBuddy -c "Set :Key value"
dist/VoiceFlow.app/Contents/Info.plist` in the CI step **before** codesign.
